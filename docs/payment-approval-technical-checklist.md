# 付款申请审批闭环：技术资料需求清单

## 1. 项目目标说明

本阶段只做“付款申请”这一条审批闭环：

付款申请提交 -> 新审批系统生成审批实例 -> AI 判断/建议 -> 人工审批 -> 审批结果回调业务系统 -> 业务系统更新付款申请状态。

范围边界：

- 当前只做付款申请审批闭环。
- 暂不接合同审批、订单利润审批、其他审批类型。
- 第一阶段以影子运行为主：新审批系统先旁路接收数据、生成建议、记录审批判断和对比结果，不直接替代钉钉。
- 当前项目中可见“资金 / 付款申请”的表单展示字段包括“付款对象、供应商/客户、业务明细、账单”，但这些字段是否等同于嘟嘟魔方、钉钉或业务系统字段，均需技术确认。
- AI 只做风险判断和审批建议，不自动替代人工审批结论；哪些字段可给 AI、哪些字段必须脱敏，需技术确认。

## 5. 测试前准备清单

### 测试账号

- 业务系统测试账号：付款申请发起人、财务/资金人员、业务经办人。具体账号需技术确认。
- 钉钉测试账号：发起人、一级审批人、二级审批人、风控审批人、抄送人。具体组织关系需技术确认。
- 新审批系统测试账号：申请人、审批人、办理人、超级管理员、AI 配置管理员。权限范围需技术确认。
- API 联调账号：业务系统调用方、审批系统回调方、查询接口调用方。鉴权方式需技术确认。

### 测试付款申请单

- 正常付款申请：字段完整、金额普通、资料齐全。
- 驳回场景付款申请：审批人明确驳回并填写原因。
- 金额较大付款申请：触发更高层级审批或风控规则，金额阈值需技术确认。
- 资料异常付款申请：附件缺失、账单异常、资金明细异常、付款对象异常等，异常定义需技术确认。
- 多账单或多明细付款申请：是否支持多条资金明细/多个账单需技术确认。
- 重复提交/重复回调付款申请：用于验证幂等机制。

### 审批场景

- 提交后新审批系统能生成审批实例，并保存业务系统 ID、审批实例 ID、钉钉相关 ID 的映射关系。
- AI 能根据允许字段生成风险判断和建议；AI 不直接改变审批结果。
- 人工审批通过后，审批结果能回调业务系统。
- 人工审批驳回后，驳回原因能回调业务系统。
- 回调失败后能按约定重试，并能防止重复更新。
- 付款申请状态和审批状态分别记录，二者映射关系需技术确认。

### 如何对比新审批系统和钉钉审批结果

- 使用同一笔付款申请的业务唯一 ID 作为对账主键，具体字段需技术确认。
- 记录钉钉审批编号、钉钉审批实例 ID、新审批系统审批实例 ID、业务系统付款申请 ID。
- 对比审批最终结果：通过、驳回、撤回、取消等状态枚举需技术确认。
- 对比审批节点：审批人、审批顺序、审批时间、驳回原因、抄送和办理节点。
- 对比 AI 建议与人工结论：记录 AI 风险等级、建议摘要、人工最终动作，不让 AI 自动替代人工。
- 对比业务系统状态：确认回调后业务系统付款申请状态是否按预期变化。

### 影子运行期间记录的数据

- 原始提交 JSON 的脱敏快照。
- 字段映射版本号和映射结果。
- 业务系统唯一 ID、钉钉审批编号、钉钉审批实例 ID、新审批系统实例 ID。
- AI 输入字段清单、AI 输出建议、风险等级、生成时间、失败原因。
- 新审批系统审批流命中的分支、审批节点、人工审批动作。
- 钉钉真实审批结果和业务系统最终状态。
- 差异记录：字段差异、审批人差异、节点差异、结果差异、状态回调差异。
- 接口调用日志：请求时间、响应时间、错误码、重试次数、幂等键、回调处理结果。

### 进入小范围试点的条件

- 已拿到付款申请原始 JSON、接口文档、状态枚举、字段映射表，并经技术确认。
- 影子运行样本覆盖正常、驳回、金额较大、资料异常等关键场景。
- 新审批系统与钉钉在审批结果、审批节点、状态流转上的差异已解释并处理。
- 回调业务系统的幂等、重试、鉴权、错误处理已联调通过。
- AI 建议的输入字段、脱敏规则、输出保存规则已确认。
- 明确试点范围：仅付款申请，限定组织/人员/金额/时间范围，其他审批类型不进入试点。
- 明确回退方案：出现异常时继续以钉钉为准，新审批系统不阻断业务。

## 6. 给技术同事的消息模板

技术同事好，我们现在先不做全部审批类型，只做“付款申请”这一条审批闭环。目标流程是：付款申请提交 -> 新审批系统生成审批实例 -> AI 判断/建议 -> 人工审批 -> 审批结果回调业务系统 -> 业务系统更新付款申请状态。第一阶段先影子运行，不直接替代钉钉。

为了开始做字段映射和接口方案，麻烦先提供/确认以下资料：

1. 付款申请提交时，嘟嘟魔方/业务系统传给钉钉或审批平台的完整原始 JSON。
2. 3-5 个真实脱敏样例 JSON，覆盖正常通过、驳回、金额较大、资料异常等场景。
3. 查询审批实例接口文档。
4. 审批结果回调业务系统 API 文档。
5. 付款申请结果处理/状态更新 API 文档。
6. 当前钉钉付款申请审批表单配置和审批流配置规则。
